FAQ Why did you do it this way?!

Q: Wow, that's a lot of output, can that be reduced?
A: Adding VERBOSE=no to localrc will tell grenade.sh to save all output to the log file as usual but only display occasional status messages to the terminal.
Q: How do I use a Gerrit branch of a project?
A: Generally this is for the trunk DevStack configuration. Add the same *_REPO and *_BRANCH lines to devstack.local.trunk that would be used for DevStack. For example, to use a specific branch of Glance: 
GLANCE_REPO=https://review.openstack.org/openstack/glance
GLANCE_BRANCH=refs/changes/11/1111/1
Q: How do I re-run just one project upgrade?
A: The upgrade-* scripts can be executed stand-alone, although the fancy output options are not available.
Q: I'd like to help!
A: That isn't a question, but please do! The source for Grenade is in github. Contributions should use the usual GitHub pull request process. This site is housed in the gh-pages branch.
Q: Why not use packages instead of DevStack?
A: Grenade is intended for testing the upgrade process throughout the development cycle and can be used both in continuous integration and by individual developers. The upgrade details learned and encapsulated in Grenade's upgrade-* scripts can be the basis for packagers to develop their packaged upgrade scripts.
Q: Why isn't $MY_FAVORITE_DISTRO supported?
A: Grenade is targeted for testing OpenStack upgrades, which tend to not be distro-sensitive. The CI system uses Ubuntu as the testing platform so Ubuntu was chosen first.
Q: What releases of Ubuntu are supported??
A: The current incarnation of Grenade supports Ubuntu Oneiric (11.10) and Precise (12.04).
Q: Does Grenade handle a multi-node installation?
A: Nope.
